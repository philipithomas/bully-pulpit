# bully-pulpit

Personal website and blog for philipithomas.com. Next.js with MDX content. Hosted on Vercel.

## Development

```bash
nvm use 24
pnpm install
pnpm dev          # Start dev server with Turbopack
pnpm build        # Production build
pnpm test         # Run Vitest tests (unit + PGlite integration)
pnpm check        # Biome lint + format check
pnpm images:optimize  # Resize oversized images, generate email variants
pnpm content:check    # Validate generated artifacts are in sync (images, search index, related)
pnpm links:check      # Validate internal links/images in content MDX (offline)
pnpm canonical:check  # Scan built HTML: every indexable page has a canonical (runs as postbuild)
pnpm search:index     # Embed new/changed chunks → search-index.json + related-posts.json
pnpm db:generate  # Generate a Drizzle SQL migration from schema changes
pnpm db:migrate   # Apply migrations to DATABASE_URL (Neon)
pnpm db:studio    # Open Drizzle Studio against the DB
```

## Architecture

- **Content**: MDX files in `content/{contraption,workshop,postcard,pages}/`. New-post pipeline: add MDX (+ cover under `public/images/covers/`) → `pnpm images:optimize` → `pnpm search:index` → commit the generated artifacts. `pnpm content:check` (offline, fast) validates all of it — email image variants exist/within budget, sources not oversized, the committed search index matches a recomputed merkle tree over the post corpus, every post has a related-posts entry — and runs in **pre-commit, CI, and the Vercel build command**, so a forgotten step fails the deploy instead of silently shipping degraded emails/pages
- **Rendering**: All public pages statically generated via `generateStaticParams`. The exception is the Printing Press admin (`/printing-press/*`), which renders dynamically: its layout awaits `requireAdmin()`, which reads the session cookie per request. **Canonical guard**: `pnpm canonical:check` (`scripts/check-canonical.ts`) scans the prerendered HTML in `.next/server/app/**/*.html` after every build and fails if any indexable page lacks exactly one `<link rel="canonical">` on the production host. It runs automatically as the `postbuild` npm hook, so it fires in pre-commit, CI, and the Vercel build with no extra wiring. The indexable/noindex partition is self-deriving from the HTML (a page that emits `<meta name="robots" content="noindex">` is exempt; the exemption is itself asserted), so the guard needs no per-route maintenance as content grows. Utility pages (`/account`, `/unsubscribe`) set `robots: { index: false }` in their metadata and are the intended exemptions
- **Auth**: Client-side overlay; subscribers/OTP/magic-link in Neon Postgres; session is a JWT (jose, HS256) in the httpOnly `bp_token` cookie (`bp_has_session` mirrors login state for the client)
- **Database**: Neon Postgres via Drizzle ORM (`src/lib/db/`). DB clients are lazy so `next build` never needs `DATABASE_URL` — keep all DB access in route handlers / server actions / workflows, never at module scope or in static generation. The `vercel.json` build command is `content:check && db:migrate:deploy && next build`: the offline content check runs first (fails fast before any state change), then migrations, then build. Migrations are **guarded to `VERCEL_ENV=production`** — preview builds must not run migrations. `db:migrate` uses `DATABASE_URL_UNPOOLED` when present. Use additive (expand/contract) migrations so a mid-deploy schema/code gap stays backward-compatible
- **Email**: AWS SES (`@aws-sdk/client-sesv2`). Transactional sends (OTP, admin notification) are synchronous; the newsletter blast runs through a durable Vercel Workflow (`src/workflows/send-newsletter.ts`). Delivery is at-least-once (no SES idempotency key). Per-recipient rows snapshot the rendered HTML + plaintext (`text_content`) so the text/plain part carries the real body. A monthly cron (`/api/cron/subscriber-backup`, first of the month at 11:00 UTC) emails admins the full subscriber list as a CSV attachment via a hand-rolled raw MIME message (`src/lib/email/mime.ts`). Bounce/complaint capture is **polling only** (deliberately no SNS/webhook — no SES-side setup): a 15-minute cron (`/api/cron/suppression-sync`) mirrors the SES account-level suppression list into `email_suppressions` and is authoritative for its source (rows SES drops are deleted); the subscribers panel surfaces when and why an address is suppressed. Every `/api/cron/*` route requires `Authorization: Bearer $CRON_SECRET`; Vercel sends that header automatically to cron paths when `CRON_SECRET` is set
- **Phone**: Twilio voice and SMS webhooks under `src/app/api/phone/` (`voice`, `recording-complete`, `recording-status`, `sms`), thin wrappers over `src/lib/phone/` — ported from junk-drawer's Rails phone backend. Auth on every webhook is a shared `?secret=` query param (`PHONE_WEBHOOK_SECRET`) embedded in the webhook URLs configured in the Twilio console, compared in constant time and failing closed when unset. The voice route speaks an AI-generated greeting (AI Gateway `anthropic/claude-haiku-4.5`, prompted with NYC time + wttr.in weather; any failure falls back to a static greeting so the call always proceeds) via Twilio `<Say>` and records up to 120 seconds; a completed recording starts the durable voicemail workflow (`src/workflows/process-voicemail.ts`): download the MP3 from Twilio → transcribe via OpenAI (`OPENAI_API_KEY` — the AI Gateway does not route transcription models) → email the transcription with the MP3 attached (the raw-MIME attachment path in `src/lib/email/mime.ts`). Recordings under 1000 bytes and empty transcriptions are skipped as butt dials. Phone notifications (missed call, voicemail, inbound SMS) go to the `ADMIN_EMAILS` allowlist (every address, via `phoneNotificationRecipients()` in `src/lib/phone/config.ts`); if the allowlist is ever blanked it falls back to the static philip@contraption.co so a notification is never dropped. **Text messaging**: inbound SMS is persisted to a `text_messages` table (`src/lib/db/queries/text-messages.ts`, deduplicated by `twilio_sid`) before the email notification fires; the Printing press Phone panel (`/printing-press/phone`) lists conversations grouped by the other party's number (newest first) and renders inbound/outbound threads, all admin-guarded. Outbound SMS and click-to-call go through `src/lib/phone/twilio.ts` (form-encoded REST against `api.twilio.com`, HTTP Basic auth with `TWILIO_SID`/`TWILIO_SECRET` — the account SID and auth token, in `accountSid:authToken` order — no-op mock off production). Click-to-call rings `OWNER_PHONE_NUMBER` first, then on answer Twilio fetches the secret-validated `POST /api/phone/connect?secret=...` callback which returns XML-escaped `<Dial callerId>` to the destination; `callerId` is allowlisted to `PHONE_NUMBERS` (the owned Twilio numbers, the 212 NYC line first). All destination numbers are E.164-validated; SMS bodies are length-bounded. Cutover: set the env vars in Vercel, point each Twilio number's voice webhook at `POST /api/phone/voice?secret=...` and its messaging webhook at `POST /api/phone/sms?secret=...`, then retire junk-drawer's `/twilio/*` routes
- **Admin (Printing Press)**: the admin panel lives at `/printing-press` (`/admin` 307-redirects there), gated by the `ADMIN_EMAILS` allowlist (`src/lib/auth/admin.ts`) on top of a normal signed-in session — page layout guard (`requireAdmin()`) + per-route `guardAdmin()` on every `app/api/printing-press/*` handler. The shell (`src/components/printing-press/`) is a single centered column: serif "Printing press" masthead over a row of plain text nav links (no sidenav, Sheet drawer, or icons; the public site header above links home). Sections: Overview, Posts (send UI), Subscribers (search by email, gravatar avatars, copy, hard-delete, suppression clearing — removes the address from the SES account-level suppression list and the local `email_suppressions` row, SES first so a failed clear leaves visible state, CSV export/import — export columns `email,name,postcard,contraption,workshop,confirmed,source,created_at`, import upserts by email and also accepts optional `source`; flag columns absent from the CSV only set defaults on new rows, so a bare email list can't re-subscribe opt-outs; `source` is backfill-only on existing rows (fills NULL or the `csv_import` placeholder, never overwrites a real captured referrer, so an attribution re-sync is idempotent); export neutralizes spreadsheet formula injection in `email`/`name`/`source`), and Phone (SMS conversation view + outbound SMS + click-to-call, see Phone above). Gravatar avatars knowingly send md5(subscriber email) to Automattic — accepted tradeoff for an admin-only list. The dropdown's "Printing Press" link shows only when `/api/auth/me` reports `isAdmin`
- **Search**: fully local, no external search service. `src/lib/search/` holds the pipeline: `corpus.ts` deterministically chunks every post (title chunk, ~1200-char body chunks broken at headings, cover-alt chunk); `pnpm search:index` embeds chunks via the AI Gateway (`openai/text-embedding-3-small` truncated to 256 dims + L2-normalized, Matryoshka-style, in `embedding.ts`) and commits them to `src/generated/search-index.json` (hashes + base64 vectors only, no text) plus `src/generated/related-posts.json` (centroid cosine similarity, score = 1 - cosine). A merkle tree over chunk hashes (`merkle.ts`) lets `content:check` verify the committed index offline and name the stale slugs; the script reuses vectors for unchanged chunk hashes so a routine run embeds only the new post. **Staleness is guarded in three layers** so the committed index never goes stale on main: (1) pre-commit and CI (`content:check`) **enforce** — a content change that did not regenerate the index fails the commit/PR with `run pnpm search:index`, no regen, no embedding call; (2) a push-to-main GitHub Action (`.github/workflows/search-index.yml`) **auto-regenerates** and commits the index back, so main self-heals even when the PR author lacked the AI Gateway key. The regen is deterministic (reuses vectors for unchanged chunk hashes), so a clean index is a no-op and the bot's own commit (marked `[skip ci]`, plus a concurrency group) does not loop. The Action needs the `AI_GATEWAY_API_KEY` Actions secret (the AI Gateway key) and `permissions: contents: write`; if main is ever branch-protected, the `GITHUB_TOKEN` push will be rejected and the bot must be allowed to bypass protection or a PAT secret supplied. Typeahead (`/api/search`) is BM25 only via MiniSearch (`lexical.ts`, title boosted 6x), in-process with no network calls; the Bell `searchPosts` tool fuses BM25 with brute-force cosine over the committed vectors via reciprocal rank fusion and degrades to BM25 if the query embedding fails
- **Chat (Bell)**: site chat sidebar (`src/components/chat/`, `app/api/chat`) streams via AI SDK + AI Gateway (`anthropic/claude-haiku-4.5` on gateway credits, no BYOK) with tools `searchPosts` (local hybrid search, see Search above) and `fetchPost`, capped at 7 steps with the last step forced to prose (`activeTools: []`). Latency: gateway routing is pinned to Anthropic direct (`order: ['anthropic']`, `sort: 'ttft'` — Bedrock/Vertex serve the same model with slower time-to-first-token), and the pre-stream checks (Firewall rate limit, BotID, body parse) run concurrently since both checks read only headers. Anthropic prompt caching is deliberately not wired: Haiku 4.5 needs a 4096-token minimum cacheable prefix and Bell's system prompt tops out around 2k tokens, so a cache marker would silently never engage. The client renders only text and tool parts, and scrubs leading tool-call JSON from text (`src/lib/chat/scrub-leaked-tool-json.ts`) — gpt-oss-120b used to leak unbound call JSON into replies; keep that guard even though the model changed
- **Styling**: Tailwind v4 CSS-first config in `src/styles/globals.css`

## Conventions

- Path aliases: `@/*` maps to `./src/*` (relative imports banned by biome)
- Biome for linting/formatting (not ESLint/Prettier)
- Single quotes, no semicolons (biome config)
- Integration tests are `*.integration.test.ts`, colocated, and run real SQL against an in-memory PGlite Postgres (no Docker/network) in the same `pnpm test` run. Harness at `src/test/integration/`: swap the Neon client with `vi.mock('@/lib/db/client', () => import('@/test/integration/db'))` (applies the real migrations, fresh DB per file, `resetDb()` per test), `session.ts` replaces `next/headers` for real-JWT sessions, `mocks.ts` has SES/BotID factories. Mock only external I/O — never `@/lib/db/queries/*`
- Fonts loaded from fonts.philipithomas.com CDN
- GitHub workflows (`.github/workflows/`): `check.yml` is CI (every push/PR to main: Biome, tsc, tests, `content:check`, `links:check`, build); `search-index.yml` regenerates and commits `src/generated/search-index.json` + `related-posts.json` on push to main (paths-filtered to `content/**`, the generator, and `src/lib/search/**`) so main self-heals after a content PR (needs the `AI_GATEWAY_API_KEY` secret and `contents: write`); `links.yml` checks external links weekly via lychee and files an issue instead of failing CI; `postcard-draft.yml` opens a draft postcard PR monthly. `claude.yml` and `claude-code-review.yml` run Claude automation
- Pre-commit hook runs lint-staged, `pnpm content:check`, and `pnpm build` — no need to build manually before pushing

## Prose style guide

ALL user-visible text follows the style guide in `content/pages/colophon.mdx` — blog posts and page copy, but also UI strings, toasts, empty states, email templates, and product naming. The rules that bite most in UI code: sentence case for titles and names ("Printing press", not "Printing Press"), no contractions, no em dashes (use periods or colons), ISO-8601 dates, active voice. The admin panel is deliberately not metrics-driven — prose summaries over stat cards; never add open/click tracking.

## Newsletters

- **Contraption** (`/contraption/`): Essays and launches. Forest (#2B4A3E) accent.
- **Workshop** (`/workshop/`): Work in progress notes. Walnut (#6B4D3A) accent.
- **Postcard** (`/postcard/`): Monthly updates. Indigo (#2C3E6B) accent.

Sending: an admin opens `/printing-press` → Posts, picks a post, previews it, sends a test to themselves, then sends to all confirmed subscribers of that newsletter. The send is a durable Vercel Workflow that enqueues per-recipient `email_sends` rows and delivers them in paced, retryable batches via SES (transient errors back off via `RetryableError`). Eligibility excludes anyone already sent or pending; the Send path heals previously-errored rows in place (`resetFailedBySlug`) so a re-send reuses them instead of duplicating. Both the Send and Retry routes gate on real workflow run liveness, not on pending `email_sends` rows: each `start()` persists its runId in `send_runs` (keyed by slug) and the guard (`isSendRunActive`, `src/lib/email/send-guard.ts`) asks the runtime `getRun(runId).status` — only `pending`/`running` returns 409. Pending rows OUTLIVE a dead run (throttle-exhausted, cancelled, crashed), so inferring "in flight" from them would deadlock the Retry resume path forever; run status distinguishes a live run from a stalled send that Retry is meant to take over. UI primitives live in `src/components/ui/` (radix + cva) on the shadcn token contract: `globals.css` defines the shadcn CSS variables (`--background`, `--primary`, `--destructive`, …) mapped to the warm palette, so `shadcn add` output drops in correctly. Rules: never run `shadcn init` (it rewrites globals.css and breaks the @font-face setup); install deps with `-w` (this repo is a pnpm workspace root); every vendored component gets an editorial pass — strip shadows and `.dark` blocks, remove rounding from interactive controls (buttons and inputs are square; `rounded-full` only for avatars and tag-style pills), sentence-case strings per the colophon, and swap stock palette utilities for warm tokens (`destructive`/site `red` for danger, `forest`/`brass` tints for status — the warm gray scale is spelled `gray-050`, not `gray-50`). A global `:focus-visible` outline in globals.css is the single keyboard focus indicator — do not add per-component focus rings. The public `.btn`/`.btn-primary`/`.btn-indigo` classes in globals.css are the marketing-CTA tier (signature arrow-reveal hover) and stay hand-built outside the token contract.
